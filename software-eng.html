Chapter 7: Software Engineering

Terms Definitions
Software Engineering The process of designing, developing, testing, and maintaining software
Integrated Development Environment (IDE) An all-in-one program used by software engineers to develop code
Association for Computing Machinery (ACM)
Purpose of ACM
Benefits of ACM Membership
Student ACM Membership		 Purpose: The world's largest scientific and educational computing society
Benefits: Acts as an umbrella organization for academic interest in computer science
Student Membership: Discounted membership for computer science students that allows access to the organization's publications
Institute of Electrical and Electronics Engineers (IEEE)
Purpose of IEEE
Benefits of IEEE Membership
Student IEEE Membership		 Purpose: To promote the development of electrical and electronic engineering and science
Benefits: Provides free access to the IEEE publications and events; helps with networking
Student Membership: Discounted membership for electrical and electronic engineering students that allows access to the organization's publications
Software Life Cycle: Development Phase
Requirements Analysis
Design
Implementation
Testing		 Requirements Analysis: Process used to determine the needs and expectations of a new software
Design: The design and function of the software is created with respect to the requirements laid out earlier in the process
Implementation: Using the software in the real world for the purpose it was intended for
Testing: Looking at the software and how it performs in order to identify bugs and inefficiencies
Modular Implementation Implementing software as a series of independent and interchangeable modules
Unified Modeling Language (UML) Official modeling system that can be used to help lay out and understand complicated software
Use Case A specific situation where a specific kind of software can be used
Use Case Diagram A visual depiction of the ways a user can interact with the system
Data Relationships:
One-to-One
One-to-Many
Many-to-Many		 One-to-One: A link between two tables that have the same information
One-to-Many: A link between two tables that have different information
Many-to-Many: When one or more items in a table have a relationship two one or more items in another table
Design Pattern A small well-defined aspect of a computer program
Software Quality Assurance (SOA) The process of testing and revising code to ensure it performs the appropriate functions
Software Testing:
Glass Box Testing
Black Box Testing
Beta Testing
Alpha Testing		 Glass Box Testing: A type of testing which shows which parts of the code were executed or not
Black Box Testing: Testing without knowing the internals of a system
Beta Testing: Allows users to try an unfinished version of a software
Alpha Testing: Allows users to try an even more unfinished version of a software
Documentation:
User Documentation
System Documentation
Technical Documentation		 User Documentation: How to use the software
System Documentation: How the software works
Technical Documentation: A document that describes the architecture or design of a software
Intelleectual Property Law:
Software License
Software Disclaimer
Best Defenses Against Software Liability Claims		 Software License: Permission to use a software by whoever designed or created it
Software Disclaimer: A list of things the software can or cannot do or be relied on for
Best Defenses Against Software Liability Claims: Carefully and thoroughly developing and testing the software

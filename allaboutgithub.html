All About GitHub
Q. What is a version control system?
A. Software that tracks and manages changes to software code.
Q. What is Git?
A. A version control system that allows multiple programmers to work on the same project together.
Q. What is GitHub?
A. It is a website that uses the version control software "Git".
Q. What is the difference between Git and GitHub?
A. Git is the version control software itself, while GitHub is the online hosting service that uses Git repositories.
Q. Who started GitHub and how was it started?
A. It was started by Tom Preston Warner, Chris Wanstrath, PJ Hyatt, and Scott Chacon in 2008.
A. It was started due to the founders wanting to create a platform for users to work on projects collaboratively.
Q. What company owns it now?
A. GitHub is owned by Microsoft.
Q. How much does as GitHub account cost?
A. GitHub is free.
Q. What is the Octocat?
A. It is the mascot of GitHub.
Git/GitHub terms:
repository: a storage location for software packages
commit: when you publish new changes to the software repository
fork: a new repository that shares code and settings with the original repository
push: the act of committing a local repository to a remote repository
pull requests: a way to tell others about changes you have pushed to a repository
workflows: an automated process that runs one or more jobs
issues: a way to discuss tasks, enhancements, bugs, etc. in a repository
raw button: a button one clicks as a way to use the raw code without styling
blame button: a button that one clicks to see a ine-by-line revision of the code.
